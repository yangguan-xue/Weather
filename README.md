知晴雨天气系统 - 完整项目流程梳理
# 一、项目核心功能
1.实时天气查询：用户查看当前城市天气<br>
2.天气预报查询：查看未来几天天气预报<br>
3.定时订阅推送：用户订阅每日定时天气推送（文本形式）<br>
4.多城市管理：支持切换不同城市<br>
5.用户管理：微信登录和订阅管理<br>

# 三、系统工作流程
## 3.1登录流程
```md
用户打开小程序
      ↓
前端调用wx.login()获取code
      ↓
发送code到后端/login接口
      ↓
后端调用微信API，换取openid和session_key
      ↓
后端生成JWT token返回前端
      ↓
前端存储token，后续请求携带
```
# 3.2获取天气流程
```md
用户进入首页或选择城市
    ↓
前端发送请求：GET /api/weather/today?city=北京
    ↓
后端验证token有效性
    ↓
检查Redis缓存中是否有该城市天气
    │
    ├─ 有缓存且未过期 → 直接返回缓存数据
    │
    └─ 无缓存或已过期
        ↓
调用和风天气API获取实时天气
        ↓
解析API返回数据，添加天气建议
        ↓
将数据存入Redis（设置5分钟过期）
        ↓
返回格式化后的天气数据给前端
```
## 3.3获取天气预报流程
```md
用户查看未来几天天气预报
    ↓
前端发送请求：GET /api/weather/forecast?city=北京&days=3
    ↓
后端调用和风天气API获取未来3天预报
    ↓
解析返回数据，格式化处理
    ↓
返回给前端展示
```

## 3.4订阅管理
```md
用户进入订阅页面
    ↓
选择城市和推送时间（如07:00）
    ↓
点击订阅按钮
    ↓
前端发送请求：POST /api/weather/subscribe
    ↓
后端验证用户，保存订阅到MySQL
    ↓
返回订阅成功信息
    ↓
前端更新UI显示已订阅状态
```

## 3.5每日定时推送
```md
Spring Schedule定时任务每天指定时间启动
    ↓
查询所有订阅用户（MySQL）
    ↓
为每个订阅用户：
    ├─ 获取对应城市的天气数据
    ├─ 生成天气文本消息
    └─ 调用微信订阅消息API发送
    ↓
用户收到微信服务通知
    ↓
点击通知进入小程序查看天气详情
```

# 四、数据库设计
```md
1. 设计数据库表结构
   - 用户表：存储微信用户信息
   - 订阅表：存储用户订阅的城市和时间
   - 天气缓存表：缓存天气数据
   
2. 创建数据库和表
   - 创建weather_db数据库
   - 执行SQL建表语句
   
3. 配置JPA实体类
   - 定义@Entity实体类
   - 配置表关系映射
   
4. 创建Repository接口
   - 继承JpaRepository
   - 定义自定义查询方法
```
# 五、第三方服务集成流程
```md
1. 申请和风天气API
```
